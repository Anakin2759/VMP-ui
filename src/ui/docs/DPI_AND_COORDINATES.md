# 高 DPI 与坐标语义

目标是把“逻辑尺寸”和“物理像素”彻底分层，解决跨屏、缩放、命中偏移和文本发糊问题。

## 一条硬规则

- 组件层、布局层、交互层统一使用逻辑坐标。
- 像素坐标仅用于渲染目标、视口裁剪和文本栅格化。

## Window Metrics（单一事实来源）

每个窗口维护：

- `windowID`
- `logicalSize`
- `pixelSize`
- `devicePixelRatio`
- `contentScale`
- `metricsDirty`

所有系统都从这份运行时度量读取窗口语义，禁止各自直接推导。

## 输入链路规范

1. 平台事件进入时保留原始像素坐标（仅调试用途）。
2. 在事件入口统一换算到逻辑坐标。
3. HitTest、State、Action 只消费逻辑坐标。

## 布局链路规范

1. 根布局尺寸使用逻辑尺寸。
2. 禁止把窗口像素尺寸写回 `Size` 等组件。
3. 滚动区域内容尺寸与命中区域一律在逻辑空间计算。

## 渲染链路规范

1. RenderContext 同时持有逻辑和像素维度。
2. 顶点与裁剪先在逻辑空间完成，再映射到像素提交。
3. 文本缓存键必须携带与 DPR 等价的尺度信息。

## 分阶段推进

1. 建立并接入 Window Metrics。
2. 完成输入统一归一化。
3. 完成布局与像素解耦。
4. 完成渲染上下文双尺寸化。
5. 完成文本缓存的 DPR 隔离。

## 验证清单

1. 100% / 150% / 200% 缩放下布局和命中一致。
2. 跨显示器拖动窗口后文本清晰且命中不偏移。
3. 滚动、文本输入、光标与选择区行为稳定。
