# UI 模块文档总览

本目录收敛为 5 份核心文档（含本索引），目标是让 UI 设计、实现、维护和演进都有单一可信来源。

## 文档清单

1. [架构锐评与边界](./ARCHITECTURE_REVIEW.md) — 模块边界、跨系统耦合点、改进方向
2. [渲染管线与性能策略](./RENDERING_PIPELINE.md) — 主链路角色与每帧流程
3. [渲染子系统锐评与优化建议](./RENDERING_ARCHITECTURE_REVIEW.md) — 现状量化分析与 P0/P1 优化项
4. [扩展指南与演进路线](./EXTENSION_AND_ROADMAP.md) — 新增控件/系统流程与路线图

## 阅读顺序

- 新同学入门：1 → 2
- 优化性能或评估渲染改动：2 → 3
- 新增控件/系统或确认下一阶段优先级：4

## 维护规则

- 新增 UI 机制时，优先更新对应主题文档，不再散落到其他目录。
- 文档以"当前实现 + 已确认方向"为准，不保留过期草案。
- 运行时入口说明以 `RuntimeFacade` 的现状为准；`Registry`/`Dispatcher` 静态接口只按兼容层理解。
- 常规帧路径说明以 `Application` 单点消费 `TaskChain` 的实现为准，不再把它写成独立公共入口。
- 所有路径引用统一使用 `src/...`（项目根下的实际目录），不要写成 `src/ui/...`。
- 所有示例都尽量绑定真实符号和目录，避免抽象口号。
- 优化建议落地后，应在 `RENDERING_ARCHITECTURE_REVIEW.md` 对应条目标注完成状态，避免重复返工。
