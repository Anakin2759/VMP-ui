## 失败详情

**测试文件**：[tests/unittest/ui/test_BatchManager.cpp](../../tests/unittest/ui/test_BatchManager.cpp#L88)（第 88-101 行）

**断言失败**（第 98 行）：
```text
Expected equality of these values:
  batchManager.getBatchCount()
    Which is: 1
  2U
    Which is: 2
```

**根因分析**：  
[src/ui/managers/BatchManager.hpp](../../src/ui/managers/BatchManager.hpp) 的 `beginBatch()` 合并条件（约第 84-105 行）仅比较 `texture` 和 `scissorRect`，未检查 `pushConstants`。  
测试场景：两个批次纹理均为 `nullptr`、裁剪区均为 `nullopt`，但 `opacity` 分别为 `1.0F` 和 `0.5F`，理应不被合并。  

**语义分歧**（需人工确认）：
- **生产代码设计**：注释明确"只比较纹理 + 裁剪区域"，且 `opacity` 等逐实体字段已在 `beginBatch` 内提取到 `m_pendingQuadParams`，后续写入顶点属性，理论上可以合并批次
- **测试期望**：不同 `opacity` 的批次不应合并，`getBatchCount()` 应为 2  
- **结论**：若 GPU 着色器从**顶点属性**读取 opacity，批次可以合并（测试预期有误）；若从 **push constants** 读取，则不同 opacity 不能合并（生产代码遗漏了 pushConstants 比较）
