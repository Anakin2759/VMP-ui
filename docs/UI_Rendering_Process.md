# PestManKill UI 模块渲染流程文档

本模块采用基于 **ECS (EnTT)** 和 **SDL3 GPU** 的高性能 UI 渲染架构。设计目标是实现流畅的 2D 渲染、灵活的布局（Yoga）以及高效的批处理（Batching）。

## 1. 核心组件总览

| 组件 | 说明 |
|------|------|
| **RenderSystem** | 顶层协调者，解析视图树并调度渲染管线 |
| **IRenderer** | 抽象接口，负责将特定的组件数据（按钮、文本等）转换为顶点数据 |
| **BatchManager** | 批次管理器，通过合并相同纹理和状态的绘制请求，最小化 Draw Call |
| **CommandBuffer** | GPU 命令抽象层，封装 SDL_GPU 的底层渲染指令 |
| **RenderContext** | 渲染上下文，在视图树递归过程中传递全局属性（Alpha、位置、裁剪区等） |

## 2. 渲染流程详解

渲染流程每帧执行一次，主要分为以下五个阶段：

### 阶段一：数据搜集 (Collection)

`RenderSystem::collectRenderData` 从根窗口实体开始，深度优先遍历 ECS 层次结构（Hierarchy）：

1. **状态合并**：叠加父节点的位置、Alpha 透明度以及缩放值。
2. **裁剪处理**：若实体包含 `ScrollArea`，则计算并推入 SDL Scissor 矩形到上下文。
3. **分发任务**：查询所有注册的 `IRenderer`。如果某个渲染器能够处理当前实体（`canHandle`），则创建一个 `RenderItem` 并存入队列。

### 阶段二：排序 (Sorting)

为了保证 UI 的遮挡关系正确：

1. **Z-Order**：根据 `ZOrderIndex` 组件对 `m_renderQueue` 进行主排序。
2. **提交顺序**：如果 Z 值相同，则保持原始遍历顺序（Stable Sort），确保后添加的子元素覆盖在父元素之上。

### 阶段三：几何组装 (Geometry Generation)

按排序后的顺序遍历 `RenderQueue`，调用各渲染器的 `collect` 方法：

- 渲染器（如 `TextRenderer` 或 `ShapeRenderer`）根据 `RenderContext` 计算最终的顶点坐标。
- 数据被提交到 `BatchManager`。

### 阶段四：批处理合并 (Batching)

`BatchManager` 在接收顶点数据时执行在线优化：

- **合并规则**：如果连续的渲染请求具有相同的 **纹理 (Texture)**、**裁剪区 (Scissor)** 和 **Shader 常量 (PushConstants)**，则合并为一个批次。
- **内存优化**：使用 `std::pmr::vector` 和预分配的内存池，减少每帧分配开销。

### 阶段五：GPU 执行 (Execution)

`CommandBuffer` 负责将最终的批次转换为 GPU 指令：

1. **资源同步**：上传顶点缓冲区（Vertex Buffer）和索引缓冲区（Index Buffer）到虚拟显存。
2. **管线切换**：设置 `SDL_GPURenderPass`，绑定着色器。
3. **绘制分发**：循环调用 `SDL_DrawGPUIndexedPrimitives`，执行批次渲染。
4. **交换缓冲**：提交命令并呈现到窗口。

## 3. 关键技术细节

### 3.1 基于 SDF 的高质量渲染

文本和图标渲染采用 **SDF (Signed Distance Field)** 技术。

- 通过 `UiPushConstants` 传递控制参数（如 `outline_width`, `edge_softness`）。
- 支持子像素平滑和无损缩放。

### 3.2 裁剪系统 (Clipping)

UI 系统支持嵌套裁剪。根 `RenderContext` 维护一个裁剪矩形栈。

- 子视图的裁剪区域是其父级裁剪区域与自身区域的交集。
- 最终坐标通过 `BatchManager` 的批次头信息传递给 GPU。

### 3.3 状态管理

- **渲染脏标记**：通过 `RenderDirtyTag` 组件标记需要重绘的部分（目前为全量重绘，用于未来局部刷新预留）。
- **缓存机制**：`PipelineCache` 缓存所有 PSO（Pipeline State Objects），`TextTextureCache` 缓存动态生成的文字贴图。

## 4. 扩展指南

若要添加新的 UI 元素类型（例如特效或复杂的自定义控件）：

1. 实现 `IRenderer` 接口。
2. 在 `canHandle` 中通过组件标签（Tag）识别目标实体。
3. 在 `collect` 中计算顶点（通常是 4 个顶点组成的 Quads）并推送到 `BatchManager`。
4. 在 `RenderSystem::initializeRenderers` 中注册该渲染器。
